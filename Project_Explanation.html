01. Overview

What is this Project?

The Dimension Extractor is a high-precision tools designed to solve a critical engineering pain point: the manual transcription of measurement data from technical PDF drawings. It transforms static drawings into structured digital data by combining modern Computer Vision, Deep Learning, and Vector Geometry.

1
Modern Pipeline

Uses a "Hybrid Engine" that perfectly parses native CAD PDFs while using Deep Learning for scanned paper drawings.

2
Interactive UX

Allows human engineers to verify and edit detection boxes on an interactive canvas before final extraction.

3
Industrial Grade

Understands complex engineering notations like Ø (diameter), ± (tolerances), and multi-line callouts.

STAR Approach

The STAR Methodology

S

Situation

Engineers manually scanning PDF drawings to extract dimensions like 25.4 mm, Ø50, or 100 ± 0.5. This process is highly inefficient, labor-intensive, and prone to human transcription errors.

T

Task

Develop a robust, high-precision automated system capable of detecting, highlighting, and digitizing dimensional data from both native CAD PDFs and legacy scanned drawings.

A

Action

Engineered a Hybrid OCR Pipeline combining PyMuPDF for vector accuracy with PaddleOCR Deep Learning for scans. Implemented a Konva.js Interactive Canvas for manual refinements and logic for tolerance parsing using multi-orientation rotation testing (0°, 90°, 180°, 270°).

R

Result

Achieved a unified platform for dimension extraction that ensures 100% confidence through Human-in-the-Loop validation, dramatically reducing manual overhead and providing clean, structured digital data.

02. Data Journey

Start-to-End Workflow

1. Intake & Standardisation
User uploads a PDF. The backend immediately converts the first page into a high-resolution (200 DPI) image using pdf2image to ensure visual consistency between the browser and the AI model.
2. Hybrid Detection (The "Smart" Part)
The system analyzes the PDF. If it's a Vector PDF (generated by CAD), it extracts text directly from the file's internal code. If it's a Scan, it activates PaddleOCR (Deep Learning) to "read" the pixels using multi-rotation detection.
3. Spatial Grouping & Validation
Isolated pieces of text like "25.0" and "±0.1" are geographically merged. Heuristics then filter out "Noise" (like company logos, title blocks, or material names) so only actual measurements remain.
4. Human Verification
The detected boxes appear on the React frontend. The user can move, resize, or add new boxes. This "Human-in-the-Loop" step ensures 100% accuracy for critical industrial tasks.
5. Structured Export
Final boxes are sent back to the server. The server crops each tiny region, performs a final "High-Focus OCR", parses the result into Nominal and Tolerance values, and generates a formatted .txt report.
03. Technology Stack

Why These Technologies?

Technology Role The "Why"
PaddleOCR Deep Learning OCR Designed for precision in industrial documents. Unlike other engines, PaddleOCR excels at reading horizontal and vertical text using advanced multi-orientation analysis (0° to 270°).
OpenCV Computer Vision Handles the "image math": cropping regions from the drawing, scaling coordinates, and drawing the visual highlights for the UI.
PyMuPDF Vector Engine Lightning-fast. It "talks" directly to the PDF's internal vector stream, allowing 100% perfect extraction without any AI errors for modern CAD files.
React + Konva Interactive UI Konva is a "Canvas" library. It allows us to render thousands of red boxes on top of a drawing while keeping the interface smooth and responsive.
Django Robust Backend Provides the secure structure needed for file handling, database persistence (MySQL), and the RESTful API endpoints.
04. Codebase Anatomy

File-by-File Breakdown

Backend: The Intelligence Core

  • services/pipeline.py
    The "Orchestrator". It decides whether to use Vector or OCR paths and coordinates the flow from image conversion to final saving.
  • services/vector_engine.py
    Uses PyMuPDF to extract text from native PDFs. Handles rotation detection and scaling text to the current image DPI.
  • services/paddle_engine.py
    The AI bridge. It initializes the Paddle AI model and processes image pixels with multi-orientation rotation support to handle vertical dimensions.
  • services/grouping_engine.py
    Geometric logic to merge adjacent text tokens. (e.g., if "20" is next to ".5", it merges them into "20.5").
  • services/tolerance_parser.py
    Strict linguistic logic. Uses Regex to split complex strings like "Ø10H7(+0.015/0)" into structured data.
  • extractor/views.py
    The API Gateway. Links the frontend requests (Upload, Detect, Extract) to the internal processing services.
  • extractor/models.py
    Database schema. Tells MySQL how to store drawing files, extracted text, and processing status.

Frontend: The User Interface

  • src/App.jsx
    The root controller. Manages the overall app state (which page are we on: Uploading, Processing, or Reviewing?)
  • src/api.js
    Centralized communication. Contains all the Axios calls that send data to the Django server.
  • components/DrawingViewer/
    The most complex component. Uses Konva.js to create the interactive PDF overlay functionality.
  • components/DataTable.jsx
    A spreadsheet-like grid for reviewing the extracted values before you download the final report.

OpenCV's Role: When you draw a box on the screen, the frontend sends {x, y, w, h}. The backend uses OpenCV to "Crop" that exact rectangle from the Original High-Res image, applies a grayscale filter to enhance contrast, and then feeds this small, clean image to PaddleOCR. This "Targeted OCR" approach is 10x more accurate than scanning the whole page at once, especially for technical fonts.
