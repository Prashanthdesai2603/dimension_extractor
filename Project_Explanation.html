Project Overview — What Is This?
Problem: Engineers manually reading PDF drawings to find dimensions like 25.4 mm, Ø50, 100 ± 0.5 is slow and error-prone. This tool automates that entire process.

This is an internal web-based tool that allows any team member to upload an engineering drawing PDF, click a button, and instantly get all dimensions detected, visually highlighted with red bounding boxes, and exported as a downloadable text file.

Upload a PDF → High-accuracy Hybrid Pipeline (Vector + Deep Learning OCR) detects all dimensions → Grouping engine handles tolerances (±) → Noise filtering removes title blocks → Results returned with red bounding boxes and downloadable list.

Hybrid Extraction Pipeline (Core Logic)

The backend/services/ system implements a 6-step high-accuracy pipeline:

1
Hybrid Mode SelectionAnalyzes PDF to detect if it's Vector (CAD) or Scanned. Selects the highest precision extraction technique.
2
Vector Extraction (PyMuPDF)Directly extracts text from PDF objects with 100% accuracy for original CAD files.
3
docTR Deep Learning (Fallback)Uses docTR (CPU mode) to perform enterprise-grade OCR on scanned or non-text PDFs.
4
Spatial GroupingCombines multi-part tokens based on baseline proximity. Handles tolerances (10.0 ± 0.1) and complex labels (4x Ø12).
5
Noise FilteringImplements spatial validation: Ignores title blocks, revision tables, and notes. Only "Dimension Patterns" pass.
6
Annotation & SaveDraws RED rectangles ONLY on validated dimensions. Saves results and returns data to frontend.

Frontend Architecture

The React app is a 3-step state machine managed in App.jsx:

📤 Step 1 — Upload

Drag-and-drop or click-to-browse PDF upload with real-time progress bar and error display. Calls POST /api/upload/.

🔍 Step 2 — Process

Shows uploaded file confirmation and a "Detect Dimensions" button. Displays animated spinner while OCR runs. Calls POST /api/process/id/.

🖼️ Step 3 — Results (Image)

Full annotated drawing displayed with RED bounding boxes around each detected dimension. Opens in new tab option.

⬇ Step 3 — Results (Export)

Scrollable list + editable textarea of all dimensions. One-click "Download dimensions.txt" button. Reset to process another drawing.

Security & Configuration
Concern How Handled
DB Credentials Stored in .env file, loaded via python-dotenv. Never hardcoded in settings.py.
File Validation Backend checks .pdf extension AND file size (< 50 MB) before saving.
CORS django-cors-headers allows only localhost:3000 in development.
Media Files Served only in DEBUG=True mode. For production → use Nginx or cloud storage.
Upload Size Limit Max 50 MB enforced at both frontend (validation) and Django (DATA_UPLOAD_MAX_MEMORY_SIZE).

How to Run the Project Locally

Prerequisites: Python 3.10+, Node.js 18+, MySQL Server, Poppler (for fallback rendering)
